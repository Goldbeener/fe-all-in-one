
LLM 
Large Language Model 
大语言模型

AI Agent
在LLM大模型的基础上，扩展了
1. 上下文召回， 收集历史信息，参与决策
2. 流程编排， 决定执行顺序
3. 工具调用，获取外部数据
4. 状态管理， 维护中间结果

普通的LLM应用强调生成回复，单次的对话，无状态，上下文关联没有或不强
AI Agent应用强调为了完成目标而组织动作


智能体不是开发出来的，而是大模型本身就具备的
我们不需要创造智能，而是给这个智能补上**持久运行的工程基础设施**

要解决的问题
+ 记忆丢失
+ 上下文溢出
+ 情绪断裂
+ 安全失控
+ 无法运营


RAG 检索增强生成，持久记忆

**数据沉淀**，根据对话信息，提取关键信息，存入外部记忆库

**向量化索引**，将关键信息向量化，转化成一组数字存入向量数据库，语意相近的文本数字相近

**语意检索**，新消息也转换成向量，与向量数据库内容对比，找到相近的历史记录，召回相关信息

**上下文组装**， 召回信息，当前信息、最近几次对话信息，拼装成完成提示词，传递给大模型

**模型生成**，大模型根据增强的提示词，生成回复

**记忆更新**，根据本次对话，产生新的记忆，更新


调度层，决定什么进prompt、什么压缩、什么长期存储、什么丢弃

**LangGraph** 负责调度决策执行哪个动作分支, 决定做什么
**LangChain** 负责具体的动作，决定怎么做
