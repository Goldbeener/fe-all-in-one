AoS Vs SoA

AoS `Array of Structures`  数组里面的每一项都是一个完整的Structure（记录/Object）
SoA `Structures of Array`  以一个对象形式存储所有数据，对象的每个字段是一条数组，数组的每一项属于同一个实体

```js
// AoS
const Points = [
	{x: 0, y: 0, z: 0},
	{x: 1, y: 1, z: 1},
	{x: 2, y: 2, z: 2},
]

// SoA
const Points = {
	x: [0, 1, 2],
	y: [0, 1, 2],
	x: [0, 1, 2],
}
```

内存布局

SoA 在内存中连续存储，有更好的CPU/缓存/计算性能；
在底层映射为ArrayBuffer

AoS 胜在可读性，常用轻量的数据存储


ArrayBuffer&TypedArray

ArrayBuffer 一块连续的、固定大小的二进制内存，是原始内存，裸内存
不能直接读写、没有类型

> 只负责存储，不知道这些内容是如何划分的，比如每个元素多少字节，因此无法读取

JS不允许随意操作裸内存，必须配合类型视图，也就是TypedArray


TypedArray = ArrayBuffer + 类型解释规则

常见的TypedArray
+ Uint8Array 每个元素1个字节
+ Int16Array 每个元素2个字节
+ Float32Array 每个元素4个字节
+ Float64Array 每个元素8个字节
