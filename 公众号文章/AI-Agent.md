# AI Agent

## 先下定义
AI Agent 是一个以**大语言模型（LLM）**为核心决策引擎，
通过感知上下文、自主规划任务、循环调用工具，在无需人工逐步干预的情况下，持续执行直至达成目标的自主系统。

用大白话来说就是：
普通 AI 是"你问它答"，每次都等你发号施令。
Agent 是"你给目标，它自己想办法完成"，会规划、会调用外部工具、会看结果、会调整、会一步步推进，直到任务完成。

用户一次提问，agent会通过与LLM的多轮的对话，更准确的回答问题。

## 实现的架构

## AI Agent 是一种架构模式，不是一种产品形态。

只要满足"LLM 自主规划 + 工具调用 + 反馈循环"这个内核，外壳可以是任何形式：
+ CLI程序
+ 桌面应用
+ 网页应用
+ APP应用
+ 浏览器插件
+ 机器人

## 结论
这样看ai-agent本质上是一个巨大的prompt工程，上下文召回、状态管理、工具调用等这些额外的能力，目的都是为了得到更充分、更完整、更详细的prompt。

所有这些机制——上下文召回、状态管理、RAG、Skills、Memory——最终落脚点确实都是构造一个**更好的输入**喂给大模型。

从这个角度看，AI Agent 确实可以理解为：一套自动化的 Prompt 构造和执行系统


### 与Prompt工程的区别是：
Prompt 工程通常是静态的——人提前设计好输入，模型输出，结束。
Agent 多出来的关键一步是**执行结果会反过来影响下一个 Prompt**：
```bash
Prompt → 大模型 → 输出 → 执行工具 → 结果 → 新的 Prompt → 大模型 → ...
```
这个闭环反馈是 Agent 和纯 Prompt 工程的本质区别。
不只是构造更好的 Prompt，而是 Prompt 在运行时会动态演化。

AI Agent 是一套动态、自演化的 Prompt 构造与执行系统，每一步的执行结果都成为下一步 Prompt 的原材料，驱动任务持续推进直到目标达成。

它不是真正的"**智能自主性**"，本质上还是**输入决定输出**。
